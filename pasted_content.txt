## 📋 Panorama Geral da Solução

O sistema será uma aplicação web full-stack chamada **"EcoMonitor Urbano"**.

* **Front-end:** Uma interface responsiva (funciona em PC e Telemóvel) onde o cidadão pode reportar problemas rapidamente.
* **Back-end:** Um servidor que processa os relatos, armazena fotos e utiliza um modelo de IA (ou lógica de classificação) para categorizar a ocorrência.
* **Painel Administrativo:** Um Dashboard visual para órgãos públicos com mapas de calor (Heatmaps) e gráficos de tendências.

## 🛠️ Instruções de Implementação

Para que este código funcione bem, aqui estão os passos que deves seguir após a IA gerar o código:

1. **Configuração do Mapa:** Vais precisar de uma "API Key" (se usares Google Maps) ou configurar o **Leaflet** (que é gratuito e excelente para este caso).
2. **Armazenamento de Imagens:** Como o projeto prevê upload de fotos, o ideal é usar um serviço como Cloudinary ou AWS S3, mas para um protótipo inicial, o armazenamento local no servidor resolve.
3. **Lógica da IA:** Para a "Classificação por IA", podes começar com uma função simples de *Keywords* (ex: se a palavra 'lixo' aparecer, classifica como 'Resíduos') ou conectar à API do Gemini/OpenAI para análise de texto.

## 📚 Documentação do Segmento de Dados

Aqui está como os dados devem ser estruturados logicamente:

| Campo | Tipo | Descrição |
| --- | --- | --- |
| `id_ocorrencia` | UUID | Identificador único do registo. |
| `latitude/longitude` | Float | Coordenadas exatas do problema. |
| `foto_url` | String | Link da imagem armazenada. |
| `status` | Enum | 'Pendente', 'Em Análise', 'Resolvido'. |
| `categoria` | String | Tipo do impacto ambiental. |
